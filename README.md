# My Plants 🪴

A small, free, low-maintenance web app to track when your houseplants need watering.

- **Home grid** of plant cards: photo, name, last-watered, and watering urgency.
- **Split "Water" button** on each card: tap to mark watered today, or open the menu for
  "yesterday / 2 / 3 days ago".
- **Add / edit plants** with a photo upload + square crop, and a per-plant watering interval.
- **Search, sort, and "needs water" filter.**
- **Installable PWA** that works offline (cached app shell + Firestore offline cache).

## Stack

- React + Vite + TypeScript, Tailwind CSS v4, React Router (HashRouter).
- **Firestore** for data — never pauses on the free tier; secured by invisible anonymous auth.
- **Cloudinary** for image hosting (unsigned uploads) — never pauses on the free tier.
- Hosted on **GitHub Pages** via GitHub Actions.

No login screen: the app signs in anonymously in the background so Firestore rules can keep
random visitors/bots out of your data.

## Local development

```bash
npm install
cp .env.example .env.local   # then fill in the values (see below)
npm run dev
```

### Environment values (`.env.local`)

All of these are **public by design** (they ship in the client bundle). Security comes from
Firestore rules + the restricted Cloudinary upload preset, not from hiding them.

**Firebase** — create a project at <https://console.firebase.google.com>:

1. Build → **Firestore Database** → create database (production mode).
2. Build → **Authentication** → Sign-in method → enable **Anonymous**.
3. Project settings → General → *Your apps* → add a Web app → copy the SDK config into the
   `VITE_FIREBASE_*` variables.
4. Deploy the rules in [`firestore.rules`](./firestore.rules) (paste into the Rules tab, or use
   the Firebase CLI).

**Cloudinary** — create an account at <https://cloudinary.com>:

1. Copy your **cloud name** → `VITE_CLOUDINARY_CLOUD_NAME`.
2. Settings → Upload → **Upload presets** → add an **unsigned** preset. Restrict it: a fixed
   folder, allowed formats (jpg/png/webp), a max file size, and an incoming transform to cap
   dimensions. Put its name in `VITE_CLOUDINARY_UPLOAD_PRESET`.

## Deployment (GitHub Pages)

1. Push to GitHub. In the repo: Settings → **Pages** → Source = **GitHub Actions**.
2. Settings → Secrets and variables → **Actions → Variables** → add all eight `VITE_*` values.
3. Make sure `base` in [`vite.config.ts`](./vite.config.ts) matches your repo name
   (`/my-plants-2/`), or `/` if you use a custom domain.
4. Push to `main` — the workflow in [.github/workflows/deploy.yml](./.github/workflows/deploy.yml)
   builds and deploys automatically.
